# 《雄安时空变迁与建设情报平台》项目升级方案与执行计划

## 一、 项目定位与升级愿景

### 1.1 从“毕业设计”到“商业落地”的跨代升级
原系统（2021年毕业设计）是一个基于 **PC端 Bootstrap + jQuery + MySQL + 商业 SuperMap iServer** 开发的静态展示系统。虽然成功完成了空间数据可视化，但在数据时效性、部署成本以及商业变现上面临瓶颈。

本项目经过重构，升级为**《雄安时空变迁与建设情报平台》**，旨在打造一个低运营成本、高商业价值的 WebGIS 变现平台：
*   **C端（情怀流量池）**：聚焦雄安数十万回迁居民对“消失的历史家园”的精神寻根与时空变迁怀旧需求。
*   **B端（刚需变现池）**：聚焦进驻雄安及周边（保定、定州）的大后方建材商、劳务商、重型机械设备商，解决他们面对全封闭工地流转快、采购信息不对称的痛点，提供**“工地商机雷达”**。

## 二、 架构丰富：轻量化与零成本 WebGIS 演进

为了彻底消除商业 GIS 引擎的高昂授权费用，同时解决 `GeoServer` 等开源 Java 地图服务器高内存开销（至少 2GB RAM 占用）的问题，我们在可行性报告的基础上进行了革命性的**“零成本轻量化”**升级设计：

```
【原毕业设计架构】                  【现代化轻量重构架构（2026）】
MySQL (存储属性)           ───► PostgreSQL 16 + PostGIS (强空间计算)
SuperMap iServer (商业GIS)  ───► 静态 GeoJSON 图层 CDN 托管 (运行成本降低为 0)
Bootstrap + jQuery (PC端)   ───► uni-app (微信小程序) + OpenLayers (移动端轻量GIS)
纯人工录入数据              ───► 4个 FastAPI AI Agent 智能体集群 (全自动更新)
```

### 2.1 零中间件 GIS 引擎方案
1.  **静态图层托管**：雄安的区划边界、老村庄原址范围等矢量数据变化频率极低，我们将其预先导出为压缩版 GeoJSON 文件，直接放入前端工程或 CDN 托管，由前端 `OpenLayers` 或微信自带 `Map` 组件解析渲染。**无需开启任何 GIS 地图服务器**。
2.  **Spring Boot 直出 GeoJSON 接口**：对于高频变动的工地打点、商机打点，使用 Spring Boot 的 JPA 查询，直接将经纬度格式化为标准 GeoJSON 数组返回给前端。

### 2.2 坐标系转换与脱敏合规设计（红线规避）
在雄安新区做地理信息系统，必须规避**非法测绘**和**保密红线**：
1.  **坐标脱敏**：AI 智能体抓取公开招投标新闻后，对项目地址进行地理编码，获取坐标后在算法中加入 `0.001 - 0.005` 度的随机偏移，实现民用级脱敏，消除涉密隐患。
2.  **合规底图**：绝不自行编制矢量底图，而是完全通过 API 引用国家法定合规的“天地图”或腾讯、高德的公开底图，仅在其上叠加商机打点（Marker）。
3.  **火星坐标系（GCJ-02）与地球坐标系（WGS-84）转换**：国内公开地图（如高德、腾讯）均采用 GCJ-02 加密坐标系，而天地图和原始 GPS 采用 WGS-84。我们将在后端集成高精度的坐标转换算法，确保打点位置分毫不差。

## 三、 AI Agent 集群深度协作机制

平台的核心壁垒在于引入了 4 个高度协同工作的 Python FastAPI AI 智能体，解决了个人独立开发中数据更新的瓶颈：

```
┌────────────────────────────────────────────────────────┐
│               1. 数据情报官 Agent                       │
│  (自动抓取雄安/保定/定州公共资源网、官方招标网新闻)  │
└──────────────────────────┬─────────────────────────────┘
                           │ 提取出不规则中文文本地址
                           ▼
┌────────────────────────────────────────────────────────┐
│               2. 地名空间解析 Agent                     │
│  (利用大模型空间推理 + Geocoding API，将地址转为GPS)  │
└──────────────────────────┬─────────────────────────────┘
                           │ 自动加上脱敏偏移，写入 PostGIS
                           ▼
┌────────────────────────────────────────────────────────┐
│               3. 商机/便民撮合 Agent                    │
│   (利用空间地理距离函数，自动为B/C端匹配最近供应链)   │
└──────────────────────────┬─────────────────────────────┘
                           │ 触发订阅消息/数据看板
                           ▼
┌────────────────────────────────────────────────────────┐
│               4. 内容营销引流 Agent                     │
│   (大模型自动生成“XX村时空变迁”短视频脚本/营销图文)   │
└────────────────────────────────────────────────────────┘
```

## 四、 数据库物理模型设计

### 4.1 重点工程/工地商机表 (`zdgc_project`)
用于存储大模型抓取、解析出的最新雄安工地和招标项目。

```sql
CREATE TABLE zdgc_project (
    id VARCHAR(64) PRIMARY KEY,
    name VARCHAR(255) NOT NULL,          -- 项目名称
    category VARCHAR(100),               -- 项目类型（绿化、安置房、基建）
    address VARCHAR(500),                -- 原始新闻中的地址
    x_longitude DOUBLE PRECISION,        -- 脱敏经度
    y_latitude DOUBLE PRECISION,         -- 脱敏纬度
    status VARCHAR(50) DEFAULT '建设中',  -- 状态 (规划中/建设中/已完工)
    introduction TEXT,                   -- 项目简介 (大模型抓取总结)
    source_url VARCHAR(500),             -- 信息源链接
    crawled_time TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

### 4.2 人文景观/老村庄变迁表 (`rwjg_nostalgia`)
用于存储被拆迁村庄的地理定位、变迁历史和情感老照片。

```sql
CREATE TABLE rwjg_nostalgia (
    id VARCHAR(64) PRIMARY KEY,
    village_name VARCHAR(255) NOT NULL,  -- 原村庄名称
    township VARCHAR(100),               -- 所属原乡镇
    current_district VARCHAR(100),       -- 现所属回迁片区
    x_longitude DOUBLE PRECISION,        -- 遗址中心经度
    y_latitude DOUBLE PRECISION,         -- 遗址中心纬度
    demolished_year INT,                 -- 拆迁年份
    history_content TEXT,                -- 村庄历史简介
    photo_gallery TEXT                   -- 情感老照片路径 (JSON 数组)
);
```

### 4.3 供应链商家表 (`supply_merchant`)
用于存储雄安周边（保定、定州、徐水等）入驻的供应商，并根据服务半径进行空间索引。

```sql
CREATE TABLE supply_merchant (
    id VARCHAR(64) PRIMARY KEY,
    merchant_name VARCHAR(255) NOT NULL, -- 商家名称
    business_type VARCHAR(100),          -- 业务类型（挖机租赁/建材批发/劳务派遣）
    contact_person VARCHAR(50),          -- 联系人
    contact_phone VARCHAR(50),           -- 电话
    address VARCHAR(500),                -- 店铺/队伍地址
    x_longitude DOUBLE PRECISION,        -- 坐标经度
    y_latitude DOUBLE PRECISION,         -- 坐标纬度
    service_radius DOUBLE PRECISION,     -- 服务半径 (单位: 公里)
    is_vip BOOLEAN DEFAULT FALSE,        -- 是否为付费VIP订阅用户
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```
